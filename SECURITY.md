# Security Policy

## MILEHIGH-WORLD LLC Security Standard

This project is a component of the **MILEHIGH-WORLD LLC: Into the Void Framework**. We take the security of our infrastructure and the integrity of the ENNEAvERSE seriously.

### Reporting a Vulnerability

If you discover a security vulnerability or a "Void Corruption" that could compromise the stability of the AeroCore, please report it immediately.

**Contact:** [milehigh.wrld@gmail.com](mailto:milehigh.wrld@gmail.com)

Please provide a detailed description of the vulnerability, including:
- The scope of the issue.
- Steps to reproduce the vulnerability.
- Potential impact on the IX-Node stabilization.

We will acknowledge your report within 24 hours and work towards a stabilization patch according to the PHNX3NT protocols.

### PHNX3NT Resilience

All security hardening and resilience work is tracked under the `phnx/` branch prefix.

## Reporting a Vulnerability

If you discover a potential security issue in this project, we ask that you notify
AWS/Amazon Security via the
[vulnerability reporting page](http://aws.amazon.com/security/vulnerability-reporting/).
Please do **not** create a public issue.

Your report should include a description of the vulnerability, the steps to
reproduce it, and its potential impact. The security team will review your
report and respond as soon as possible.

## Security Updates

We take the security of our project seriously. When a security vulnerability is
reported and confirmed, we will work to provide a fix and release an updated
version of the extension. We encourage users to keep their extension updated to
the latest version to ensure they have the latest security patches.
